第十四章项目沟通管理

学习建议与考情分析

  1. 学习建议：项目沟通管理属十大管理内容，在上午选择题、案例分析、论文写作中均有考查。沟通管理和干系人部分上午约考4 - 6分，沟通部分约2 - 3分。项目经理大部分时间用于沟通，沟通管理在项目各阶段都很重要，案例分析中问题较多，需重点掌握。

  2. 考情分析

    • 2023年05月：上午选择题3分，案例分析题6分，论文写作0分，总计9分。
    • 2023年11月：机考上午选择题3分左右，案例分析题第1批5分、第2批10分，论文写作0分，每个批次总计10分左右。
    • 2024年05月：机考上午选择题3分左右，案例分析题0分，论文写作0分，总计3分。

十大知识领域与沟通管理ITO

十大知识领域

包括整合管理、范围管理、进度管理、成本管理、质量管理、资源管理、沟通管理、风险管理、采购管理、干系人管理。

沟通管理ITO

规划沟通管理

  • 输入：项目章程、项目管理计划（资源管理计划、干系人参与计划等）、项目文件（需求文件、干系人登记册等）、事业环境因素、组织过程资产。

  • 工具和技术：专家判断、沟通需求分析、沟通技术、沟通模型、沟通方法、人际关系与团队技能（沟通风格评估、政策意识、文化意识）、数据表现（干系人参与度评估矩阵）、会议。

  • 输出：沟通管理计划、项目管理计划（更新）、项目文件（更新，如干系人登记册、项目进度计划） 。

管理沟通

  • 输入：项目管理计划（资源管理计划、沟通管理计划、干系人参与计划）、项目文件（变更日志、问题日志等）、工作绩效报告、事业环境因素、组织过程资产。

  • 工具和技术：沟通技术、沟通方法、沟通技能（沟通胜任力、反馈、非口头技能、演示）、项目管理信息系统、项目报告、人际关系与团队技能（积极倾听、冲突管理等）、会议。

  • 输出：项目沟通记录、项目管理计划（更新）、项目文件（更新）、组织过程资产（更新）。

监督沟通

  • 输入：项目管理计划（资源管理计划、沟通管理计划、干系人参与计划）、项目文件（问题日志、经验教训登记册等）、工作绩效数据、事业环境因素、组织过程资产。

  • 工具和技术：专家判断、项目管理信息系统、数据表现（干系人参与度评估矩阵）、人际关系与团队技能（观察/交谈）、会议。

  • 输出：工作绩效信息、变更请求、项目管理计划（更新）、项目文件（更新）

管理基础

沟通

项目沟通管理包括制定有效沟通策略和执行沟通活动。沟通是通过多种方式发送或接收信息，形式有书面、口头、正式或非正式等。

沟通模型

  • 关键要素：编码（将思想转化为他人能理解的语言）、信息和反馈信息、媒介（传递信息的方法）、噪声（干扰信息传输和理解的因素）、解码（把信息还原成有意义的思想或想法，易发生误解）。
  • 基本状态：已发送、已收到、已理解、已认可、已转化为积极的行动，其中让对方转化为积极行动最难。

沟通分类

按范围分为内部沟通和外部沟通；按方式分为正式沟通（报告、正式会议等）和非正式沟通（电子邮件、社交媒体等）；按层级分为层级沟通 - 垂直（向上、向下）和层级沟通 - 水平（横向）；按性质分为官方沟通（年报等）和非官方沟通；按形式分为书面沟通和口头沟通。

沟通技巧

  • 有效沟通属性：沟通目的明确、了解接收方需求及偏好、监督并衡量沟通效果。
  • 书面沟通5C原则：正确的语法和拼写、简洁的表述、清晰的目的和表述、连贯的思维逻辑、善用控制语句和承接。
  • 项目经理沟通技巧：积极倾听、理解文化和个人差异、识别并管理干系人期望、强化技能。

管理新实践

将干系人纳入项目评审范围；邀请干系人参加项目会议；增多使用社交工具；采用多面性沟通方法

沟通管理过程

过程概述

规划沟通管理

基于干系人信息需求等制定沟通方法和计划，作用是及时提供信息、引导干系人参与、编制书面沟通计划，整个项目期间定期开展。

管理沟通

确保项目信息有效收集、生成、发布等，促进项目团队与干系人之间的信息流，整个项目期间开展。

监督沟通

确保满足项目及其干系人的信息需求，优化信息传递流程，整个项目期间开展。

与5大过程组关系

规划沟通管理在规划过程组，管理沟通在执行过程组，监督沟通在监控过程组。

裁剪考虑因素

需考虑干系人、物理地点、沟通技术、语言、知识管理等因素进行裁剪。

敏捷与适应方法

在模糊项目环境中，简化信息获取通道，经常团队检查，集中办公，透明发布项目成果，定期邀请干系人评审。

规划沟通管理详细内容

输入

项目章程、项目管理计划、项目文件（需求文件、干系人登记册）、事业环境因素、组织过程资产。

工具与技术

  • 专家判断：借助专家经验和知识。
  • 沟通需求分析：确定干系人的信息需求，包括类型、格式和价值。
  • 沟通技术：如对话、会议等，选择受信息需求紧迫性等因素影响。
  • 沟通模型：有线性、互动性等多种形式。
  • 沟通方法：互动沟通（实时多向信息交换）、推式沟通（向特定接收方发送信息）、拉式沟通（接收方自行访问信息） 。
  • 人际关系与团队技能：沟通风格评估、政策意识、文化意识。
  • 数据表现：干系人参与度评估矩阵。
  • 会议：用于沟通和决策。

输出

  • 沟通管理计划：涵盖干系人沟通需求、信息内容、上报步骤等多方面内容。

  • 项目管理计划（更新）：根据沟通规划调整相关计划。

  • 项目文件（更新）：如干系人登记册等文件更新。

管理沟通详细内容

输入

项目管理计划、项目文件（变更日志、问题日志等）、工作绩效报告、事业环境因素、组织过程资产。

工具与技术

  • 沟通技术：选择合适技术传递信息。
  • 沟通方法：根据情况选择互动、推式或拉式沟通。
  • 沟通技能：沟通胜任力、反馈、非口头技能、演示。
  • 项目管理信息系统：辅助信息管理。
  • 项目报告：编制和发布项目信息。
  • 人际关系与团队技能：积极倾听、冲突管理等。
  • 会议：促进沟通和解决问题。

输出

项目沟通记录、项目管理计划（更新）、项目文件（更新）、组织过程资产（更新）。

监督沟通详细内容

输入

项目管理计划、项目文件（问题日志、经验教训登记册等）、工作绩效数据、事业环境因素、组织过程资产。

工具与技术

  • 专家判断：获取专业意见。
  • 项目管理信息系统：分析信息。
  • 数据表现：利用干系人参与度评估矩阵调整沟通。
  • 人际关系与团队技能：观察和交谈发现问题。
  • 会议：讨论和决策。

输出

工作绩效信息、变更请求、项目管理计划（更新）、项目文件（更新）。

练习与真题

练习

涵盖沟通模型、书面沟通原则、沟通管理过程作用等知识点的选择题和判断题。

历年真题练习

2023 - 2024年的真题，涉及有效沟通属性、规划沟通描述、项目报告、沟通管理计划制定时间、监督沟通措施等内容

第十五章项目风险管理

学习建议与考情分析

  1. 学习建议：项目风险管理是十大管理内容之一，在上午选择题、案例分析、论文写作中均有考查。上午一般考查3分，在案例分析中出现次数较多，且常考工具技术，需重点学习。
  2. 考情分析
    • 2023年05月：上午选择题3分，案例分析题10分，论文写作75分，总计88分。
    • 2023年11月：机考上午选择题3分左右，案例分析题第4批20分，论文写作0分，总计23分。
    • 2024年05月：机考上午选择题3分左右，案例分析题0分，论文写作0分，总计3分。

十大知识领域与风险管理ITO

  1. 十大知识领域：涵盖整合管理、范围管理、进度管理、成本管理、质量管理、资源管理、沟通管理、风险管理、采购管理、干系人管理。
  2. 风险管理ITO
    • 规划风险管理：输入包括项目章程、项目管理计划等；工具和技术有专家判断、数据分析等；输出为风险管理计划。
    • 识别风险：输入涉及项目管理计划、项目文件等；工具和技术包含专家判断、数据收集等；输出有风险登记册、风险报告等。
    • 实施定性风险分析：输入含项目管理计划、项目文件等；工具和技术有专家判断、数据收集等；输出是项目文件（更新） 。
    • 实施定量风险分析：输入有项目管理计划、项目文件等；工具和技术包括专家判断、数据收集等；输出为项目文件（更新）。
    • 规划风险应对：输入包括项目管理计划、项目文件等；工具和技术有专家判断、数据收集等；输出有变更请求、项目管理计划（更新）等。
    • 实施风险应对：输入涉及项目管理计划、项目文件等；工具和技术有专家判断、人际关系与团队技能等；输出为变更请求、项目文件（更新）。
    • 监督风险：输入含项目管理计划、项目文件等；工具和技术有数据分析、审计等；输出有工作绩效信息、变更请求等。

管理基础

  1. 项目风险概述：项目风险是不确定事件或条件，对项目目标有正面或负面影响，既包含威胁也有机会，源于项目中的不确定因素，在项目不同阶段有所不同，早期不确定性最大。
  2. 风险的属性
    • 随机性：风险事件发生及其后果具有偶然性。
    • 相对性：风险对不同主体影响不同，人们的风险承受能力受收益大小、投入多少、项目活动主体地位和资源等因素影响。
    • 可变性：包括风险性质、后果变化以及出现新风险。
  3. 风险的分类
    • 按风险后果划分：纯粹风险无获利可能，投机风险可能获利也可能损失，二者在一定条件下可相互转化。
    • 按风险来源划分：分为自然风险和人为风险（行为、经济、技术、政策、组织等方面）。
    • 按风险是否可管理划分：可管理风险可预测并控制，不可管理风险在一定条件下可转化为可管理风险。
    • 按风险影响范围划分：局部风险影响范围小，整体风险影响范围大。
    • 按风险后果承担者划分：涉及项目业主、政府、承包商等多种主体的风险。
    • 按风险可预测性划分：已知风险后果可预见，可预测风险可预见发生但后果不可预见，不可预测风险难以预见。
  4. 风险成本及其承担：风险成本包含有形成本（直接损失和间接损失）、无形成本（减少机会、阻碍生产率提高、造成资源分配不当）以及预防与控制风险的成本（投保、咨询等费用）。
  5. 管理新实践
    • 变异性风险：由规划事件等方面的不确定性导致，可通过蒙特卡洛分析处理。
    • 模糊性风险：因对未来不确定、知识不足影响项目目标达成，可借助外部专家等方式处理。
    • 项目韧性：用于应对突发性风险，可通过预留储备等手段加强。
    • 整合式风险管理：在适当层面管理风险，利用组织级方法确保风险管理一致性，创造整体价值。

风险管理过程

  1. 过程概述
    • 规划风险管理：定义风险管理活动实施方式，确保风险管理与项目风险程度及对组织和干系人的重要性相匹配，在项目立项阶段开始并早期完成，可能在后期重新开展。
    • 识别风险：识别单个和整体项目风险来源并记录特征，是迭代过程，项目各阶段风险会变化，需全员参与。
    • 实施定性风险分析：评估风险发生概率、影响等特征进行优先级排序，为后续分析或行动提供基础，关注高优先级风险，在项目全生命周期定期开展，敏捷开发中每次迭代前进行。
    • 实施定量风险分析：对已识别风险和不确定性来源对项目目标的影响进行定量分析，适用于大型复杂等项目，也可在规划风险应对后开展，分析应对措施有效性。
    • 规划风险应对：制定应对风险的可选方案、策略和行动，应对方案需与风险重要性匹配，考虑次生风险和应急计划。
    • 实施风险应对：执行商定的风险应对计划，由风险责任人负责，主动管理项目风险。
    • 监督风险：跟踪已识别风险，识别分析新风险，评估风险管理有效性，依据绩效信息判断各项风险管理相关情况。
    • 与5大过程组关系：规划风险管理、识别风险、实施定性风险分析、实施定量风险分析、规划风险应对在规划过程组；实施风险应对在执行过程组；监督风险在监控过程组。
  2. 裁剪考虑因素：裁剪风险管理需考虑项目规模、复杂性、重要性和开发方法等因素。
  3. 敏捷与适应方法：敏捷或适应型方法通过审查增量工作产品、加快知识分享管理风险，迭代期选择工作内容时考虑风险，迭代期间识别、分析和管理风险。

规划风险管理详细内容

  1. 输入：项目章程、项目管理计划、项目文件、事业环境因素、组织过程资产。
  2. 工具与技术
    • 专家判断：借助专家经验和知识。
    • 数据分析：采用干系人分析法确定干系人风险偏好。
    • 会议：用于沟通和决策。
  3. 输出 - 风险管理计划
    • 风险管理策略：描述管理项目风险的一般方法。
    • 方法论：确定风险管理的具体方法、工具和数据来源。
    • 角色与职责：明确风险管理活动中各角色的职责。
    • 资金：确定风险管理活动资金及应急、管理储备使用方案。
    • 时间安排：确定风险管理过程的实施时间和频率。
    • 风险类别：借助风险分解结构（RBS）构建。
    • 干系人风险偏好：针对项目目标表述为可测量的风险临界值。
    • 风险概率和影响：根据项目环境等制定。
    • 概率和影响矩阵：计算风险概率 - 影响分值，排列风险优先级。
    • 报告格式：确定风险管理结果的记录、分析和沟通方式。
    • 跟踪：确定风险活动记录和审计方式。

识别风险详细内容

  1. 输入：项目管理计划、项目文件、采购文档、协议、事业环境因素、组织过程资产。
  2. 工具与技术
    • 专家判断：获取专业意见。
    • 数据收集：头脑风暴获取风险来源清单；核查单基于历史信息编制，用作提醒但不能取代风险识别工作；访谈资深参与者等识别风险。
    • 数据分析：根本原因分析找问题深层原因；假设条件和制约因素分析识别威胁和机会；SWOT分析检查项目优劣势、机会和威胁；文件分析审查项目文件识别风险。
    • 人际关系与团队技能：确保风险识别工作有效进行。
    • 提示清单：依据风险分解结构底层类别识别单个项目风险。
    • 会议：促进风险识别。
  3. 输出
    • 风险登记册：记录已识别风险清单、潜在风险责任人、潜在风险应对措施。
    • 风险报告：说明整体项目风险来源和已识别单个项目风险概述信息。
    • 项目文件（更新）：相关文件因风险识别工作更新。

实施定性风险分析详细内容

  1. 输入：项目管理计划、项目文件、事业环境因素、组织过程资产。
  2. 工具与技术
    • 专家判断：依靠专家经验。
    • 数据收集：通过访谈收集数据。
    • 数据分析：风险数据质量评估评价数据准确性和可靠性；风险概率和影响评估考虑风险发生可能性和对项目目标的影响；其他风险参数评估考虑紧迫性等特征。
    • 人际关系与团队技能：引导提高定性分析有效性。
    • 风险分类：对风险进行分类。
    • 数据表现：概率和影响矩阵将风险概率和影响映射，划分风险优先级；层级图用于多参数风险分类。
    • 会议：用于风险分析和讨论。
  3. 输出 - 项目文件（更新）
    • 假设日志：记录假设条件相关信息。
    • 问题日志：更新问题相关内容。
    • 风险登记册：更新风险概率、影响评估、优先级等信息。
    • 风险报告：记录重要风险、风险优先级列表和结论。

实施定量风险分析详细内容

  1. 输入：项目管理计划、项目文件、事业环境因素、组织过程资产。
  2. 工具与技术
    • 专家判断：获取专业指导。
    • 数据收集：通过访谈收集数据。
    • 人际关系与团队技能：引导风险研讨会，更好收集数据。
    • 不确定性表现方式：用概率分布表示活动持续时间等的不确定性。
    • 数据分析：模拟（如蒙特卡洛分析）评估风险综合影响；敏感性分析确定关键风险因素；决策树分析选择最佳行动方案；影响图辅助决策，分析要素影响。
  3. 输出 - 项目文件（更新） - 风险报告
    • 整体项目风险评估结果：评估整体风险可能性。
    • 项目详细概率分析结果：呈现详细概率分析情况。
    • 单个项目风险优先级清单：列出单个项目风险优先级。
    • 定量风险分析结果趋势：展示分析结果的变化趋势。
    • 风险应对建议：提出应对风险的建议。

规划风险应对详细内容

  1. 输入：项目管理计划、项目文件、事业环境因素、组织过程资产。
  2. 工具与技术
    • 专家判断：借助专家经验制定应对策略。
    • 数据收集：通过访谈制定应对措施。
    • 人际关系与团队技能：引导提高应对策略制定有效性。
    • 威胁应对策略：上报超出项目范围或权限的威胁；规避消除威胁或降低发生概率至零；转移将应对责任转移给第三方；减轻降低风险概率或影响；接受低优先级或难以应对的威胁。
    • 机会应对策略：上报超出项目范围或权限的机会；开拓确保高优先级机会出现；分享转移机会应对责任并分享收益；提高增加机会出现概率和影响；接受低优先级或难以应对的机会。
    • 应急应对策略：针对有预警信号的风险制定特定条件下执行的应对计划。
    • 整体项目风险应对策略：规避严重负面影响的整体风险；开拓显著正面影响的整体风险；转移或分享组织难以应对的整体风险；减轻或提高整体风险级别；接受无法主动应对的整体风险。
    • 数据分析：备选方案分析比较应对方案；成本收益分析评估应对策略成本有效性。
    • 决策：多标准决策分析对风险应对策略排序。
  3. 输出
    • 变更请求：因风险应对策略调整提出变更。
    • 项目管理计划（更新）：更新相关计划。
    • 项目文件（更新）：记录应对策略、行动、触发条件等信息，更新风险登记册和风险报告。

实施风险应对详细内容

  1. 输入：项目管理计划、项目文件、组织过程资产。
  2. 工具与技术
    • 专家判断：获取专业支持。
    • 人际关系与团队技能：运用影响力促使风险责任人采取行动。
    • 项目管理信息系统：辅助风险应对管理。
  3. 输出
    • 变更请求：实施过程中可能产生变更。
    • 项目文件（更新）：记录风险应对实施情况。

监督风险详细内容

  1. 输入：项目管理计划、项目文件（问题日志、经验教训登记册等）、工作绩效数据、工作绩效报告。
  2. 工具与技术
    • 数据分析：技术绩效分析比较技术成果与计划；储备分析评估剩余应急储备合理性。
    • 审计：风险审计评估风险管理过程有效性。
    • 会议：定期召开风险审查会检查风险应对有效性。
  3. 输出
    • 工作绩效信息：反映风险应对效果。
    • 变更请求：根据监督结果提出变更。
    • 项目管理计划（更新）：调整相关计划。
    • 项目文件（更新）：更新风险相关文件。
    • 组织过程资产（更新）：积累风险管理经验。

风险管理顺序及练习真题

  1. 风险管理顺序：识别风险、实施定性风险分析、实施定量风险分析（可选择）、规划风险应对、实施风险应对、监督风险。
  2. 本章练习：涵盖风险分类、风险识别依据、预期货币价值计算等知识点的选择题和判断题。
  3. 历年真题练习：2023 - 2024年真题，涉及识别风险、风险分析技术、整体项目风险应对策略、定量风险分析技术等内容。

Web3详解
本篇是本人对于Web3研究的总结 结合各方面的研究资料 包括亚马逊 Amazon AWS官网对于WEB3的理解，播客夜航西飞中的解释 什么是Web3？Web3是区块链等技术的总称，这些技术可以分散互联网上的数据所有权和控制权。大多数互联网应用程序都由集中式实体控制，这些实体决定如何保存和使用最终用户数据。Web3（也称为Web 3.0、去中心化Web或语义Web）技术允许社区驱动的项目，而不是集中式管理结构。在这些项目中，最终用户控制数据、确定定价、直接为技术开发做出贡献，并在项目的方向上拥有更重要的发言权。这些技术具有自动调节用户相互交互方式的机制。因此，不需要集中式实体来管理这些交互。 Web 3.0的核心理念是什么？Web 3.0具备四项主要功能。 去中心化去中心化的Web应用程序是Web 3.0的关键功能。其目的是在去中心化网络中分发和存储数据。在这些网络中，不同的实体拥有底层基础设施，用户直接向存储提供商付费以访问该空间。 去中心化的应用程序还将信息副本存储在多个位置，并确保整个过程中的数据一致性。每位用户可以控制其数据存放的位置，而不必将其移交给集中式基础设施。去中心化的互联网用户可根据需要出售自己的数据。 去信任性在集中式Web应用程序和服务中，用户通常需要信任中央权威机构来管理其数据、交易和交互。这些中央权威机构可以控制用户数据，并且可以操纵系统的规则。数据可能存在安全风险或管理不善，从而导致用户信息丢失或滥用。 相比之下，Web3引入去信任性，因此用户可以在无需信任任何特定方的情况下进行交易和交互。 语义网借助语义网，应用程序能够通过理解Web数据的内容和上下文来执行复杂的任务。语义网使用元数据和人工智能为用户生成的数据提供含义（语义）。 Web 3.0旨在更全面地转向目前存在于现有Web技术某些方面中的语义网技术。例如，搜索引擎可提供更准确且与上下文相关的搜索结果，而智能代理则可帮助用户更高效地执行任务。 互操作性Web 3.0的目标是在不同技术之间建立更多的互连，从而数据无需中介即可在不同平台之间流动。互操作性使数据具有可移植性，因此用户可以在服务之间无缝切换，同时保持自己的首选项、配置文件和设置。 与此同时，集成各种物联网（IoT）设备的协议将Web的覆盖范围扩展到传统边界之外。例如，支持无边界交易的加密货币技术允许跨地域和政治边界进行价值交换。 为什么Web 3.0如此重要？当互联网刚刚起步时，仅限阅读的Web体验很常见。最终用户只能阅读相关公司发布的内容，这些公司已购买并维护托管静态网页的基础设施。 随着博客和社交媒体平台等Web...
