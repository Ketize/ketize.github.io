Web3详解

Created2025-03-03|Updated2025-03-26|Invest-Crypto
|Post Views:

本篇是对Web3研究的总结

结合各方面的研究资料

资料来源包括不限于亚马逊 Amazon AWS官网、池建强播客夜航西飞、LinkedIn Community、Medium    

什么是Web3？

Web3是区块链等技术的总称，这些技术可以分散互联网上的数据所有权和控制权。

大多数互联网应用程序都由集中式实体控制，这些实体决定如何保存和使用最终用户数据。

Web3（也称为Web 3.0、去中心化Web或语义Web）技术允许社区驱动的项目，而不是集中式管理结构。

在Web3项目中，最终用户控制数据、确定定价、直接为技术开发做出贡献，并在项目的方向上拥有更重要的发言权。这些技术具有自动调节用户相互交互方式的机制。因此，不需要集中式实体来管理这些交互。

集中式、分散式、分布式

集中式应用程序

我们每天使用的大多数互联网应用程序都是中心化的，它们由特定的公司或个人拥有，他们提供并维护在计算机、服务器甚至集群上执行的源代码。

中心化应用是工程师习惯构建、用户习惯使用的大多数应用。

数据存放在由公司控制的集中数据库中。你最终必须相信这家公司在做正确的事情，并且符合你的利益。

去中心化应用程序

分布式意味着计算分布在多个节点上，而不是一个节点。去中心化意味着没有节点会指示其他节点该做什么。

代码在对等节点网络上运行，没有任何单个节点可以控制 dApp。根据 dApp 的功能，可以使用不同的数据结构来存储应用程序数据。比特币使用区块链去中心化交易账本。

分布式应用程序

计算分布在各个组件中的应用程序通过传递消息来沟通和协调其操作。组件之间相互交互以实现共同目标。

Web1、Web2、Web3

Web1 是一个只读网络，用户只能消费内容。Web2 是一个参与式网络，允许用户创建和共享内容。

Web 3.0的核心理念是什么？

Web 3.0具备四项主要功能。

去中心化

去中心化的Web应用程序是Web 3.0的关键功能。其目的是在去中心化网络中分发和存储数据。在这些网络中，不同的实体拥有底层基础设施，用户直接向存储提供商付费以访问该空间。

去中心化的应用程序还将信息副本存储在多个位置，并确保整个过程中的数据一致性。每位用户可以控制其数据存放的位置，而不必将其移交给集中式基础设施。去中心化的互联网用户可根据需要出售自己的数据。

去信任性

在集中式Web应用程序和服务中，用户通常需要信任中央权威机构来管理其数据、交易和交互。这些中央权威机构可以控制用户数据，并且可以操纵系统的规则。数据可能存在安全风险或管理不善，从而导致用户信息丢失或滥用。

相比之下，Web3引入去信任性，因此用户可以在无需信任任何特定方的情况下进行交易和交互。

语义网

借助语义网，应用程序能够通过理解Web数据的内容和上下文来执行复杂的任务。语义网使用元数据和人工智能为用户生成的数据提供含义（语义）。

Web 3.0旨在更全面地转向目前存在于现有Web技术某些方面中的语义网技术。例如，搜索引擎可提供更准确且与上下文相关的搜索结果，而智能代理则可帮助用户更高效地执行任务。

互操作性

Web 3.0的目标是在不同技术之间建立更多的互连，从而数据无需中介即可在不同平台之间流动。互操作性使数据具有可移植性，因此用户可以在服务之间无缝切换，同时保持自己的首选项、配置文件和设置。

与此同时，集成各种物联网（IoT）设备的协议将Web的覆盖范围扩展到传统边界之外。例如，支持无边界交易的加密货币技术允许跨地域和政治边界进行价值交换。

为什么Web 3.0如此重要？

当互联网刚刚起步时，仅限阅读的Web体验很常见。最终用户只能阅读相关公司发布的内容，这些公司已购买并维护托管静态网页的基础设施。

随着博客和社交媒体平台等Web 2.0技术的出现，应用程序变得更具交互性。可以生成和发布内容或与他人交换服务。但是，所有交互都由中央第三方权威机构管理，这些权威机构从服务交换中获得商业利益。这些权威机构还可以拥有和控制最终用户创建的数字资产。

例如，集中式自由职业者平台将自由职业者与客户联系起来，而租房平台将业主与租房者联系起来。服务提供商和服务用户都会创建服务配置文件、服务描述、用户个人资料、博客、视频和评论等数据。这些平台集中管理所有这些数据。

Web 2.0面临的挑战

虽然中央平台促进和规范双方之间的交互，但Web 2.0机制带来若干挑战：

  • 服务提供商可能无法在不损失声誉和丢失客户群体的情况下将其数据转移到其他平台。
  • 服务用户对其数据使用和管理方式的控制权有限。
  • 集中式平台可能会独立制定某些对最终用户产生重大影响的决策。例如，这些平台可以筛选某些用户生成的内容或限制最终用户对某些网站功能的访问。

Web 3.0的优点

Web 3.0旨在转向读取/写入/拥有模式，即数据创建者拥有和控制自己的数据，并在如何使用和管理数据方面拥有更重要的发言权。Web 3.0技术提供多种机制，因此最终用户不仅仅是客户，而是成为股东和参与者。接下来介绍其他一些优点。

  • 提高参与度：用户以更有意义的方式在彼此之间和与解决方案提供商进行交互。他们会因为积极参与在线社区而获得激励，而无需请求其交换数据。
  • 增强隐私：个人用户决定谁有权访问他们的数据。存放数据的基础设施的所有者无法访问个人用户的数据。您的在线交互可能公开可见，但您的身份保持机密性。
  • 民主化沟通：Web 3.0旨在消除地理、政治和企业沟通障碍。Web 3.0限制大型科技公司的审查权，并在安全要求和更高的透明度之间取得平衡。

Web 3.0中有哪些主要技术？

区块链技术构成了许多Web 3.0应用程序的基础，提供了透明度、不可变性和去信任性。区块链是去中心化的分布式分类账，用于存储节点网络上的交易记录或数据。

区块链数据库将数据存储在按时间顺序保持一致的区块中，只有在通过对等网络达成共识后，才能删除或修改这些区块。区块链内置的机制可以阻止未经授权的交易条目并在这些交易的共享视图中创建一致性。因此，可以使用区块链创建不可更改或不可变的分类账，以此跟踪所有类型的交易。

接下来将介绍其他推动Web 3.0发展的关键技术。

令牌化

可以通过令牌化扩展区块链技术应用程序。令牌化是在区块链上将现实世界或数字资产表示为数字令牌的过程。

这些令牌是所有权、访问权或其他形式价值的加密表示。例如，可以用令牌代表实物和数字资产，例如房地产、股票、商品、艺术品、音乐，甚至是游戏内物品。

每个令牌可以代表底层资产的特定部分或整个单位，使其可分割且易于交易。虚拟世界中有不同的令牌类型，例如受证券监管约束的证券令牌，或代表不允许分形所有权的独特、不可分割资产的不可替代令牌（NFT）。

WebAssembly

WebAssembly（Wasm）是一种用于堆栈式虚拟机的二进制指令格式。它在浏览器内的沙盒环境中运行，这意味着其无法访问用户的本地文件系统。

WebAssembly支持在Web浏览器中运行高性能代码，这为去中心化应用程序在不同平台上的高效运行奠定了基础。开发人员可以按照接近原生的速度运行代码，与JavaScript等传统Web技术相比，显著提高了性能。

语义网技术

借助语义网技术，应用程序能够更全面地理解和解释客户数据。这些技术使用关联数据原理将多个数据集相互关联或在Web上发布结构化数据。下面提供一些示例。

资源描述框架

使用资源描述框架（RDF），可以将语句以主语-谓词-对象的形式表示为三元组。这些三元组创建基于图形的数据结构，用来表示不同实体的关系。SPARQL是一种用于查询RDF数据的查询语言。

Web本体语言

Web本体语言（OWL）用于定义本体或知识和概念之间关系的形式表示。可以使用OWL指定类、属性和实例，以及促进推论和推理。

Web 3.0应用程序有哪些示例？

各种区块链技术应用程序正在构建中，并将作为API和服务公开。然后，它们用来针对不同的使用案例构建其他Web3应用程序。下面提供一些示例。

智能合约

智能合约是自我实现的合约，其预定义规则以代码编写而成。当满足某些条件时，这些合约会自动执行协议条款。例如，如果合约规定物品的所有权转让给任何为该物品支付费用的人员，则您只需向该合约发送要价。然后，分类账通过使用下一个区块中的交易来自动更新，以反映您成为新的所有者。

去中心化自治组织（DAO）是一种商定的智能合约，可自动对资源池（令牌）进行去中心化决策。拥有令牌的用户对资源的使用方式进行投票，并且代码会自动得出投票结果。

可以使用智能合约为抵押贷款、债券和证券等金融工具编程条件（也称为去中心化金融）。或者，可以通过供应链促进货物的跟踪和付款。智能合约可以免除使用中介，这有助于提高交易的效率和安全性。

去中心化识别

去中心化身份是一种旨在让个人对其数字身份拥有更多控制权和所有权的技术。在传统的在线系统中，用户通常依靠集中式身份提供商（如社交媒体平台或电子邮件服务）来管理其身份和访问各种在线服务。

去中心化身份将身份信息的控制权转移回个人用户。该技术使用与去中心化标识符（DID）文档关联的全球唯一标识符。此文档包含与身份相关的公有密钥、加密材料和服务端点。该技术使用选择性披露来共享特定的身份属性，并最大限度地减少在身份验证过程中对个人身份信息（PII）的暴露。

区块链

区块链主链基础

区块链运作原理与特性

区块链技术将众多服务器相连，当合法修改一台机器数据时，数据会像病毒一样扩散到链上每台机器；

而非法修改会因少数服从多数原则被拒绝，这体现了其不可篡改性，保障个人数据安全

若想摧毁区块链，需同时毁掉所有服务器，否则它能自行恢复

常见主链及其分层结构

常见的区块链主链有以太坊主链、Banis主链、Solana主链等，它们处于layer1层，是web3世界的基础架构，

类似web2世界的微信、抖音等平台,layer2层是对layer1层的扩展，例如Optimistic能提升以太坊主网交易速度、降低gas费；

layer3层用于开发智能合约，类似微信小程序、抖音小程序在web2世界的角色

gas费的产生机制

以太坊主链中，发起人邀请拥有大量服务器的人（矿工）加入，用原生币ETH作为报酬

当开发的智能合约在主链上进行数据改动（如转账等交易信息变更）时，需向矿工支付gas费，而查询数据无需付费

智能合约开发详解

开发语言与工具选择

以太坊智能合约开发使用solidity语言，有在线编辑器remix，方便开发者在网页上编写、部署和调试代码；

Solona链则采用rust语言开发智能合约

智能合约开发无需独立数据库，可将以太坊主网当作数据库使用

遵循协议与潜在骗局

以太坊智能合约需遵循erc20协议，实现特定函数功能才能成为标准erc20并发布到主网

但开发过程中存在诸多骗局，如授权骗局，骗子通过诱导用户点击领取空头，实则申请获取用户web3钱包授权，进而操纵钱包代币；

助记词骗局中，用户注册钱包时的助记词若泄露，不法分子会利用钱包原生币授权，监测并盗走原生币

原生币与代币的区别

每个主链都有自己的原生币，如以太坊的ETH、Solana的SOL、Binance的BNB，原生币可用于支付gas费，是链内交易的一般等价物

USDT是代币，由泰达公司发行，宣称每发行一个USDT就存储一美元，为虚拟世界原生币在现实中赋予价值，帮助用户规避主链生态波动风险

合约功能拓展与部署流程

除erc20协议要求的函数，开发者还可添加预售、质押、空投等功能

代码完成后进行部署，先在虚拟环境测试，再连接钱包部署到测试网或主网

测试网有水龙头可获取测试用原生币，主网部署则需支付真实的ETH

DAPP前端搭建步骤

前端开发技术运用

使用VoE代码开发DAPP前端，借助Metmask插件（Metmask SDK，即MM SDK），通过复制合约的ABI（类似web2的API）到JS中，实现前端与后端智能合约的交互

DAPP功能实际演示

以哈利币配套DAPP为例，用户在钱包中添加哈利币合约后，可进行预售、质押、赎回、空投等操作

其中，涉及数据改变的操作（如质押、赎回、领取空投）需支付gas费

加密货币交易所剖析

交易所类型对比

加密货币交易所分为中心化和去中心化两种

中心化交易所如欧易、币安，面向大众，操作简单，用户量多，但需实名认证（KYC）；

去中心化交易所如UNISWAR，无需繁琐手续，不过对用户专业性要求更高，用户量相对较少

虚拟币上线交易所流程

在去中心化交易所UNISWAR上线虚拟币，需依次点击“池”“新的头寸”，选择合适选项，设置ETH与代币的交易对及初始价格，存入交易对资金（存入后无法取出）

加密货币投资风险警示

人们热衷于加密货币，是因为其进入交易所后类似无监管的股票，上链和去去中心化交易所类似可作假刷量的IPO

币圈作假成本低，普通投资者易被虚假热度误导，投资风险极大，需谨慎对待

第十四章项目沟通管理学习建议与考情分析 学习建议：项目沟通管理属十大管理内容，在上午选择题、案例分析、论文写作中均有考查。沟通管理和干系人部分上午约考4 - 6分，沟通部分约2 - 3分。项目经理大部分时间用于沟通，沟通管理在项目各阶段都很重要，案例分析中问题较多，需重点掌握。 考情分析 2023年05月：上午选择题3分，案例分析题6分，论文写作0分，总计9分。 2023年11月：机考上午选择题3分左右，案例分析题第1批5分、第2批10分，论文写作0分，每个批次总计10分左右。 2024年05月：机考上午选择题3分左右，案例分析题0分，论文写作0分，总计3分。 十大知识领域与沟通管理ITO十大知识领域包括整合管理、范围管理、进度管理、成本管理、质量管理、资源管理、沟通管理、风险管理、采购管理、干系人管理。 沟通管理ITO规划沟通管理 输入：项目章程、项目管理计划（资源管理计划、干系人参与计划等）、项目文件（需求文件、干系人登记册等）、事业环境因素、组织过程资产。 工具和技术：专家判断、沟通需求分析、沟通技术、沟通模型、沟通方法、人际关系与团队技能（沟通风格评估、政策意识、文化意识）、数据表现（干系人参与度评估矩阵）、会议。 输出：沟通管理计划、项目管理计划（更新）、项目文件（更新，如干系人登记册、项目进度计划） 。 管理沟通 输入：项目管理计划（资源管理计划、沟通管理计划、干系人参与计划）、项目文件（变更日志、问题日志等）、工作绩效报告、事业环境因素、组织过程资产。 工具和技术：沟通技术、沟通方法、沟通技能（沟通胜任力、反馈、非口头技能、演示）、项目管理信息系统、项目报告、人际关系与团队技能（积极倾听、冲突管理等）、会议。 输出：项目沟通记录、项目管理计划（更新）、项目文件（更新）、组织过程资产（更新）。 监督沟通 输入：项目管理计划（资源管理计划、沟通管理计划、干系人参与计划）、项目文件（问题日志、经验教训登记册等）、工作绩效数据、事业环境因素、组织过程资产。 工具和技术：专家判断、项目管理信息系统、数据表现（干系人参与度评估矩阵）、人际关系与团队技能（观察/交谈）、会议。 输出：工作绩效信息、变更请求、项目管理计划（更新）、项目文件（更新） 管理基础沟通项目沟通管理包括制定有效沟通策略和执行沟通活动。沟通是通过多种方式发送或接收信息，形式有书面、口头、正式或非正式等。 沟通模型 关键要素：编码（将思想转化为他人能理解的语言）、信息和反馈信息、媒介（传递信息的方法）、噪声（干扰信息传输和理解的因素）、解码（把信息还原成有意义的思想或想法，易发生误解）。 基本状态：已发送、已收到、已理解、已认可、已转化为积极的行动，其中让对方转化为积极行动最难。 沟通分类按范围分为内部沟通和外部沟通；按方式分为正式沟通（报告、正式会议等）和非正式沟通（电子邮件、社交媒体等）；按层级分为层级沟通 - 垂直（向上、向下）和层级沟通 -...
